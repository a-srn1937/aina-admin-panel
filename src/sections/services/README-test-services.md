# مدیریت سرویس‌های آزمون

## تغییرات اعمال شده

### قابلیت‌های جدید در تب سرویس‌های آزمون:

✅ **افزودن سرویس جدید**
- دکمه "افزودن سرویس" در بالای لیست
- فرم مخصوص ایجاد سرویس برای آزمون
- `test_id` خودکار پر می‌شود

✅ **ویرایش سرویس**
- آیکون ویرایش برای هر سرویس
- فرم ویرایش کامل

✅ **حذف سرویس**
- آیکون حذف برای هر سرویس
- تأیید قبل از حذف

## کامپوننت‌های جدید

### TestServiceCreateForm
```jsx
<TestServiceCreateForm
  testId={testId}
  onSuccess={() => {
    // callback after successful creation
  }}
/>
```

**ویژگی‌ها:**
- `test_id` خودکار پر می‌شود
- فرم ساده‌تر (بدون انتخاب تست)
- مناسب برای ایجاد سرویس در محیط آزمون

### TestServicesList (بروزرسانی شده)
```jsx
<TestServicesList testId={testId} />
```

**قابلیت‌های جدید:**
- دکمه افزودن سرویس
- بروزرسانی خودکار لیست پس از تغییرات
- UI بهتر با header

## مقایسه با صفحه سرویس‌ها

| قابلیت | صفحه سرویس‌ها | سرویس‌های آزمون |
|---------|---------------|------------------|
| مشاهده همه سرویس‌ها | ✅ | ❌ (فقط سرویس‌های آزمون) |
| ایجاد سرویس | ✅ (با انتخاب تست) | ✅ (خودکار برای تست) |
| ویرایش سرویس | ✅ | ✅ |
| حذف سرویس | ✅ | ✅ |
| فیلتر و جستجو | ✅ | ❌ |

## مزایای این رویکرد

1. **UX بهتر**: کاربر نیازی نداره بین صفحات جابجا بشه
2. **سرعت بیشتر**: همه کارها در یک محیط
3. **کمتر خطا**: `test_id` خودکار پر می‌شه
4. **منطقی‌تر**: مدیریت سرویس‌ها در محل مربوطه

## نحوه استفاده

1. **برای افزودن سرویس جدید:**
   - برو به تب "سرویس‌ها" در جزئیات آزمون
   - کلیک روی "افزودن سرویس"
   - فرم رو پر کن (test_id خودکار پر میشه)
   - "ایجاد سرویس" رو بزن

2. **برای ویرایش سرویس:**
   - آیکون ویرایش رو بزن
   - تغییرات رو اعمال کن
   - ذخیره کن

3. **برای حذف سرویس:**
   - آیکون حذف رو بزن
   - تأیید کن
